# 现实优先判定协议 v1.0

## 1. 元前提

- 连续性是协作基石（invariant），不是优化目标。
- 人与模型都不具备预知能力。
- 决策必须以可验证现实为主输入，而不是历史壳叙事。
- `Shell` 优先级不得高于当下现实约束。

## 2. 判定标准

当出现以下任一条件时，判定为 `壳优先偏差`（即虚假限制高风险）：

1. 使用历史身份/历史成功作为主要依据，缺少当下可验证事实。
2. 新现实输入已变化，但决策结论不更新。
3. 论证中大量“我们一直这样/我就是这样”而缺少资源与底线参数。

## 3. 执行规则

1. 先刷新现实约束：钱、时间、能量、底线、可用资源上限。
2. 再读取壳：只用于责任接口与稳定交付，不用于直接压制新流。
3. 若壳与流冲突：强制澄清，禁止默认沿用历史路径。

## 4. 干预分级（防对抗）

1. 默认低干预：给一条现实校验问句，不阻断执行。
2. 高风险强干预：仅当不可逆高风险命中（如 L4 gate）才要求二次确认。
3. 可退出：用户说“知道了”或表现不耐烦，立即降级为纯执行。
4. 禁止人格评判：不使用“你在逃避/你被壳控制”之类表述。

## 5. 输出要求

每次关键建议必须显式给出：

- `现实依据`（可验证事实）
- `壳依据`（历史接口/承诺）
- `冲突处理`（为何保留或为何更新）

## 6. 与虚假限制关系

- 现实优先不是“反历史”，而是“反历史霸权”。
- 壳夺权是虚假限制的重要来源：它把过去有效路径伪装成永恒正确。
